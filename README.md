# 见己

> 本地优先的 AI 自我复盘工作台

见己帮你整理个人材料、回看重要经历、发现反复出现的模式，并生成阶段性的自我画像。它适合做日常复盘和个人整理，但不提供心理诊断，也不能替代心理咨询或医疗建议。

## 现在可以做什么

- 选择一个本地文件夹作为长期工作区。
- 随时上传 `.md`、`.txt`、`.csv`、`.json`，也可以直接写一段近况。
- 通过六个引导问题完成复盘，生成新的阶段画像，同时保留旧版本。
- 创建多个聊天会话，并搜索、改名、归档或从列表中移除会话。
- 为每个会话单独选择画像版本、参考材料和最近消息数量。
- 查看回答引用的材料名称和原文摘录。
- 生成阶段复盘和个人 SWOT 报告。

## 数据放在哪里

- 应用只会写入所选文件夹里的 `ai-self-analysis/`。
- 导入材料时会复制一份，原文件不会被删除、移动、改名或覆盖。
- 从列表中删除会话属于软删除，原始 JSON 记录仍保留在本地。
- API Key 保存在当前浏览器的本地存储中，不会写进工作区。
- 生成画像、聊天或报告时，相关文字会发送给你配置的模型服务。本项目没有产品后端，也不会把材料另存到自己的服务器。

工作区大致如下：

```text
你选择的文件夹/
└── ai-self-analysis/
    ├── materials/
    ├── profiles/
    ├── sessions/
    ├── reports/
    └── index/
```

## 浏览器要求

完整功能依赖 File System Access API。建议使用桌面版 Chrome 或 Edge，并通过 HTTPS 或本地开发地址访问。

Firefox、Safari 和部分移动浏览器可以打开页面，但可能无法选择并持续写入本地文件夹。移动端界面已经适配，更适合查看和继续对话；首次选择工作区仍建议在桌面浏览器完成。

## 本地运行

建议使用 Node.js 22。

```bash
npm install
npm run dev
```

运行测试和生产构建：

```bash
npm test
npm run build
```

## GitHub Pages

仓库包含 [GitHub Pages 部署工作流](.github/workflows/deploy-pages.yml)。推送到 `main` 后，工作流会按照仓库名设置 Vite base path，并发布 `dist/`。

第一次部署时，需要在仓库的 `Settings -> Pages` 中把 `Source` 设置为 `GitHub Actions`。启用后，默认访问地址是：

<https://baokker.github.io/local-self-reflection/>

不要把个人 API Key 放进 GitHub Secrets 或构建环境。每位使用者都应在自己的浏览器中配置模型。

## DeepSeek 浏览器连接

静态网页能否直接调用模型，取决于模型服务是否允许浏览器跨域请求。部署完成后，请先在设置页点击“测试连接”。如果浏览器拦截请求，页面本身仍然能用，但模型相关功能需要通过可信的本地代理或桌面应用运行。不要使用公开代理转发个人材料和 API Key。

2026 年 7 月 13 日，本项目曾在本地浏览器中使用 `deepseek-v4-pro` 完成连接测试，接口返回 HTTP 200。GitHub Pages 上线后仍需从最终 HTTPS 地址重新测试。
